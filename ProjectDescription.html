Project Description

Overview

The Herb Gardenz project is a community-based herb garden website and app that allows people to find, pin and share locations of herb gardens and herbs. The website allows users to upload photos of herbs and locations for others to see as well as commenting on locations. This allows for the community aspect of the service to thrive as the service relies on users for its functionalities. Motivation for this project was that we wanted to create a service that was community based as well as informative and hopefully will allow people to save time and money using the website. Hopefully, an outcome of using the app is a strengthened community and with recent events we believe that an app like Herb Gardenz a promising idea. The app also features a list of herbs with information about each for if a user wants to look at expanding their knowledge or wanting to try something new, the information includes things such as flavor profiles and recopies that use the herb.

A goal for this project is to further community-based ideas in applications and promote the use of community features. The Herb Gardenz website relies on having an active community and such promoting this will help the site. The site will have a map and GPS feature that allows users to see where they are in relation to herbs that have been imputed by other users. They are also able to click and see the information of the herb from the pin on a location. We plan to use the Google API for the map and GPS features.

Our motivations for this project are to create an app using community-based features and demonstrate our knowledge of IT systems. The purpose of the website is to help other save money, live more sustainably and promote a healthier lifestyle using community gardens to help with their cooking. The site will use the Google API for the map services and to implement this we will use Java. Java is a skill that is sought after by some employers with this programming language being popular with many websites, mobile apps, GUI applications and more. Skills in HTML and CSS are also shown with the website and shows knowledge of design and more with an employer. This app follows the trend of using app services with many applications using a GPS and map features.

While there are many gardening services and apps to help with cooking, our website aims to have the users search for community gardens and other places to find herbs and we are trying to have our website be informative and community based unlike many competitors. There are many competitors that an “pin” locations for others to see but I have not been able to find a service that focuses on our topic and purpose, similar apps are for joke services or things that cannot be found via google maps, and that is a similar goal that we share and unlike our competitors, we allow users to review and test different locations imputed by another user, with possibly the ability to have fake locations or old locations removed

For the project there are simple yet integral features which the website must have. The key features of our product include:

  • An index system to house and share data about specific plants that are sought after. This library shares a basic description of the plants, an image for reference and situations in which the plants can be used. The information about the plants will be gathered from a collective information website such as Wikipedia and other scholarly articles which will be used to give a brief description. The index will be on a separate page than the maps page. A search function built into the index to easily find the plant or herb that the user requires.
  • A map feature in which people can input their location, the name of the plant and a category based on the website filters. After the user inputs the location, anyone on the website would be able to view the plants and the location that its in. They would then be able to go and pick it up. The map feature is implemented using JavaScript, which uses “OpenLayers” and the open-source community map “OpenStreetMap”.
  • We will have a feature for users to provide feedback and information on the website so that we can further update and improve in the future. This will be used for people to voice their concerns and share their opinions.

    • To save our data, we plan to use a database which would store both the images of the herbs we have in the index/library and the information stored for these herbs. The choice for this would be MongoDB as it is best for storing the images we need and data that we need. Since it is a NoSQL database, we will need to learn more about how to manage and create this database.

A platform’s tool and technology stack can be divided into three sections: Hardware, Firmware, and Software. Hardware to decide how the application will be hosted, Firmware to power the hardware and to run the Software application that’ll be pushed to users.

Server:

We initially hosted the platform on an old laptop (HP Pavillion x360) that was repurposed into a home server as the Lead Developer, Shivam, had experience in self-hosting. However, this quickly changed after realizing the issue of future maintenance, reliability, and scaling issues. To resolve this, we switched to using the Platform as a Service (PaaS) solution Heroku. Heroku allows us to automatically deploy the latest version of our application as it is synced with our organization Git repository. We can migrate this deployment to Amazon Web Services in the future as our userbase scales.

Firmware: As Heroku is a PaaS, we lost control over what backend firmware the server operates on, in future deployments on AWS, we plan on using an EC2 instance running Debian 11.3 (Linux). Both Hamish and Shivam have previous experience with using Linux operating systems and would be suitable for maintaining this possible deployment.

Application:

HerbGardenz is a web platform that aims to be accessible for browsers across the desktop and mobile market with the aim of providing a simple community contributed database of herbs all over the world.

As a web platform, we have used the latest stable technologies seen in the industry such as Node.JS 18.2 to serve as our platform’s JavaScript runtime environment and use OpenLayer 6.14 to create an interactive map component to display API data from the open-source community map ‘OpenStreetMap.’ These technologies are wrapped in a frontend powered by basic HTML5, CSS3 and the CSS framework Tailwind CSS v3. Extra UI component customization is powered through daisyUI v2.15.1. NodeJS and the accompanying used libraries are new technologies that all team members have no experience in. As the platform scales, it is planned to “containerize” the application through Docker 20.10, this makes it easy to deploy and facilitates the transfer between Heroku to AWS. The application uses features implemented in versions released in the past, however, by using the latest stable versions, we can ensure that our platform has the latest security updates and bug fixes.

Our production runtime is deployed through Vite 2.9.9, a standard in node software stacks.
